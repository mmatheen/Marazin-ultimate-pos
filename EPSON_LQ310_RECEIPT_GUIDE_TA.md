# Epson LQ-310 டாட் மேட்ரிக்ஸ் பிரிண்டரில் POS ரசீது அச்சிடுதல் – தமிழ் வழிகாட்டி

இந்த வழிகாட்டி **முதல் முறையாக** டாட் மேட்ரிக்ஸ் பிரிண்டரில் ரசீது அச்சிடுபவர்களுக்காக எழுதப்பட்டுள்ளது.  
காகித அகலம்: **9.5 அங்குலம்**, பயன்படுத்தக்கூடிய உள்ளடக்கம் அகலம்: **8.5 அங்குலம்** (இருபுறமும் டிராக்டர் துளைகள் 0.5 அங்குலம்).

---

## 1. Epson LQ-310 பிரிண்டர் செட்அப் (காகித அளவு, மார்ஜின், ஃபான்ட், வரி இடைவெளி)

### 1.1 விண்டோஸில் பிரிண்டர் சேர்க்கவும்
- **Settings → Devices → Printers & scanners** செல்லுங்கள்.
- **Add a printer or scanner** கிளிக் செய்து Epson LQ-310 ஐ தேர்ந்தெடுக்கவும் (USB/LPT).
- தேவைப்பட்டால் Epson இணையதளத்திலிருந்து LQ-310 டிரைவர் பதிவிறக்கி நிறுவவும்.

### 1.2 காகித அளவு (Paper Size)
- **Settings → Devices → Printers → Epson LQ-310 → Manage → Printing preferences** திறக்கவும்.
- **Paper Size** / **Page Setup** பிரிவில்:
  - **Custom** அல்லது **User-defined** தேர்ந்தெடுத்து:
    - **Width: 9.5 in** (அல்லது 241 mm)
    - **Height:** உங்கள் ரசீது நீளத்திற்கு ஏற்ப (எ.கா. **11 in** அல்லது தேவைப்படும் அங்குலம்).
  - அல்லது **A4** போன்ற விருப்பம் இருந்தால், அகலத்தை **9.5 in** ஆக மாற்ற முடிந்தால் அதைப் பயன்படுத்துங்கள்.

### 1.3 மார்ஜின்கள் (Margins) – டிராக்டர் துளைகளைத் தவிர்க்க
- **Margins** / **Page Setup** பிரிவில்:
  - **Left margin: 0.5 in** (இடது டிராக்டர்)
  - **Right margin: 0.5 in** (வலது டிராக்டர்)
  - **Top / Bottom:** தேவைக்கு ஏற்ப (எ.கா. 0.25 in).
- இதனால் உள்ளடக்கம் **8.5 அங்குலம்** அகலத்திற்குள் மட்டுமே அச்சிடப்படும்; டிராக்டர் துளைகளில் ச overlap வராது.

### 1.4 ஃபான்ட் மற்றும் லைன் ஸ்பேசிங் (பிரிண்டர் பண்புகள்)
- **Printing preferences** இல்:
  - **Font:** **Courier New** அல்லது **Epson** மோனோஸ்பேஸ் ஃபான்ட் (டாட் மேட்ரிக்ஸ்க்கு ஏற்றது).
  - **Quality:** **Draft** அல்லது **Normal** (வேகத்திற்கு Draft சிறந்தது).
- **Line spacing** பெரும்பாலும் பிரிண்டர் டிரைவரில் நேரடியாக இல்லை; அதை **பிரௌசர்/HTML பக்கத்தில்** CSS `line-height` மூலம் கட்டுப்படுத்துவோம் (படி 3).

### 1.5 சுருக்கம்
| அமைப்பு        | மதிப்பு        |
|----------------|----------------|
| Paper Width    | 9.5 in         |
| Left Margin    | 0.5 in         |
| Right Margin   | 0.5 in         |
| Usable Width   | 8.5 in         |
| Font (பரிந்துரை) | Courier New / monospace |

## 2. Blade டெம்ப்ளேட்டை 8.5 அங்குல பயன்படுத்தக்கூடிய அகலத்திற்கு மாற்றுதல்

### 2.1 செய்ய வேண்டிய மாற்றங்கள்
- **@page** மற்றும் **பிரிண்ட் பிரிவில்** பக்க அகலத்தை **8.5 in** ஆக வைக்கவும் (உள்ளடக்கம் மட்டுமே 8.5 in; மார்ஜின்கள் பிரிண்டர் செட்அப்பில் 0.5+0.5 கொடுக்கப்பட்டிருப்பதால் மொத்த காகிதம் 9.5 in).
- **html, body** மற்றும் **.invoice-page** `width: 8.5in` ஆக இருக்கட்டும்.
- **Padding:** இட/வல பக்கங்களில் கூடுதல் படி தவிர்க்க **0.15in–0.2in** போதுமானது (பிரிண்டர் ஏற்கனவே 0.5 in மார்ஜின் கொடுப்பதால்).

### 2.2 குறிப்பு
- டெம்ப்ளேட்டில் இப்போது `size: 8.0in 11in` இருந்தால் அதை **8.5in** ஆக மாற்றியுள்ளோம்; பிரிண்டர் மார்ஜின்கள் 0.5+0.5 in என செட் செய்தால் உள்ளடக்கம் 8.5 in பகுதியில் சரியாக வரும்.

## 3. டேபிள் நெடுக்கள், ஃபான்ட், லைன்-ஹைட் – டாட் மேட்ரிக்ஸ்க்கு ஏற்றமைப்பு

### 3.1 ஃபான்ட்
- **Courier New**, **Consolas** அல்லது **monospace** பயன்படுத்துங்கள். எழுத்துக்கள் சம அகலமாக இருப்பதால் நெடுக்கள் சீராக வரும்.
- Font size: **10px–12px** (டாட் மேட்ரிக்ஸ் தெளிவுக்கு 10–11px பரிந்துரை).

### 3.2 லைன் ஹைட் (வரி இடைவெளி)
- `line-height: 1.2` முதல் `1.35` வரை (குறைவாக இருந்தால் வரிகள் ஒட்டிக்கொள்ளும்; அதிகமாக இருந்தால் காகிதம் வீணாகும்).
- டெம்ப்ளேட்டில் `line-height` ஏற்கனவே `$finalLineHeight` உடன் செட் செய்யப்பட்டுள்ளது; டாட் மேட்ரிக்ஸ்க்கு **1.25** அல்லது **1.3** போன்று சோதித்துப் பாருங்கள்.

### 3.3 டேபிள் நெடுக்கள்
- அனைத்து நெடுக்களின் **மொத்த அகலம் 8.5 in** க்குள் இருக்க வேண்டும்.
- **SN** சிறியதாக (0.25–0.3 in), **Item** அகலமாக, **Qty/Price/Amount** வலது சீரமைப்பு.
- `table-layout: fixed` பயன்படுத்தினால் நெடுக்கள் நிலையான அகலத்தில் இருக்கும்; டாட் மேட்ரிக்ஸில் சீராக அச்சிடும்.

### 3.4 பாடிங்
- `padding: 2px 4px` போன்ற சிறிய மதிப்புகள்; அதிகமான பாடிங் அச்சில் இடம் வீணாக்கும்.

## 4. விரும்பினால் &lt;pre&gt; பிளாக்கள் – முழுமையான சீரமைப்புக்கு

HTML டேபிள்கள் சில பிரிண்டர்/டிரைவர் காம்போக்களில் நெடுக்களை சரியாக அச்சிடாவிட்டால், **monospace + &lt;pre&gt;** பயன்படுத்தலாம்.

### 4.1 யோசனை
- ஒவ்வொரு வரியையும் **ஒரு நிலையான எழுத்து எண்ணிக்கை** (எ.கா. 80 அல்லது 96 கேரக்டர்கள்) கொண்ட வரியாக வடிவமைத்து, இடது/வலது சீரமைப்புக்கு **spaces** பயன்படுத்துங்கள்.
- ஃபான்ட் **Courier New**, `font-size: 10px` அல்லது 11px, `line-height: 1.25`.

### 4.2 எடுத்துக்காட்டு (பொருளின் வரி)
```html
<pre class="receipt-line" style="font-family: 'Courier New', monospace; font-size: 10px; line-height: 1.25; margin: 0;">{{ sprintf('%2d %-20s %6s %8s %8.2f', $sn, substr($itemName, 0, 20), $qty, $amount, $total) }}</pre>
```
- `sprintf` மூலம் நெடுக்களை நிர்ணயித்து, வெள்ளை இடங்களால் இடம் நிரப்பலாம்.

### 4.3 எப்போது பயன்படுத்துவது
- முதலில் **HTML டேபிள்** மூலம் அச்சிட்டுப் பாருங்கள். நெடுக்கள் குழம்பினால் மட்டும் **&lt;pre&gt;** அணுகுமுறைக்கு மாறுங்கள்.

## 5. டெஸ்ட் அச்சு மற்றும் சீரமைப்பு – டிராக்டர் துளைகளைத் தவிர்த்து

### 5.1 முதல் டெஸ்ட்
1. பிரிண்டர் மார்ஜின்கள் **Left 0.5 in, Right 0.5 in** என செட் செய்து ஒரு ரசீதை அச்சிடுங்கள்.
2. அச்சான காகிதத்தை பாருங்கள்: உள்ளடக்கம் **இட/வல டிராக்டர் துளைப் பகுதியில்** (0.5 in) சென்று overlap ஆகிறதா?

### 5.2 இடது/வலது வெளியேறினால்
- **இடது** பக்கம் துளைப் பகுதிக்குள் சென்றால்: பிரிண்டர் **Left margin** ஐ **0.5 in** இலிருந்து **0.55 in** அல்லது **0.6 in** ஆக அதிகரிக்கவும் (அல்லது Blade இல் left padding சிறிது அதிகரிக்கவும்).
- **வலது** பக்கம் வெளியேறினால்: **Right margin** ஐ 0.55 in / 0.6 in ஆக அதிகரிக்கவும்.

### 5.3 அகலம் சரிபார்ப்பு
- டெம்ப்ளேட்டில் `.invoice-page { width: 8.5in; }` உள்ளதா என பார்க்கவும்.
- பிரௌசர் **Print Preview** (Ctrl+P) இல் பக்கம் 8.5 in அகலத்தில் தெரிகிறதா என சரிபார்க்கவும்.

### 5.4 வரி இடைவெளி / ஃபான்ட்
- வரிகள் ஒன்றோடொன்று ஒட்டினால்: CSS இல் `line-height` ஐ 1.3 அல்லது 1.35 ஆக உயர்த்துங்கள்.
- எழுத்து பிரிகிறது/படிக்க கடினமானால்: ஃபான்ட் சைஸை 11px அல்லது 12px ஆக உயர்த்துங்கள்; அகலம் 8.5 in க்குள் இருக்க டேபிள் நெடுக்களைக் குறைக்கவும்.

### 5.5 சுருக்கம்
| சிக்கல்              | செயல்                                      |
|----------------------|---------------------------------------------|
| இடது துளை overlap   | Left margin 0.55–0.6 in அல்லது left padding ↑ |
| வலது துளை overlap   | Right margin 0.55–0.6 in அல்லது right padding ↑ |
| வரிகள் ஒட்டுதல்    | line-height 1.3–1.35                        |
| எழுத்து தெளிவின்மை | font-size 11–12px, table column width குறைக்கவும் |
